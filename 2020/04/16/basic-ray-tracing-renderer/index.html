A Basic Ray Tracing Renderer

There is this github project ‘Tinny Ray Tracer’, which introduces the ray tracing under 300 lines code. It was a great fun to follow the instructions and made the programme.

I try to summarize some extended notes and resources together for further learning.

Preparation

All we need is C++ and a bit theory of ray tracing, no other fancy libraries. We can dive into the core directly without any configuration of OpenGL and other libraries. In order to keep it simple, we will render the image into a file, instead of using OpenGL to draw in the window.

To make the programming with multi-dimensional vectors easy, we would better to define some templates for vectors.

You can skip this part and just include “geometry.h”. It is not essential, for implementing the main part.

Step 1: Draw an image with colors

Let’s simply paint some color first, and ouput it to a pixmap file. We use PPM. It is a very simple format for pixmap.

The file format looks like this:

P6        // this is the format name
1024 768  // height and width of the image
255       // use 256-color
0 0 0 120 120 0 .... // RGB color number
